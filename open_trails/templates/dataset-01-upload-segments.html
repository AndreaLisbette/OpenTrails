ZIP up your shapefile

Upload a zipped shapefile that describes your trail network. This data should be a 'polyline' file that represents the actual paths a trail takes.

    As you probably know, a "shapefile" is actually a set of several files. To use this converter, we require the following files be included in a single zipped upload:

  • .shp
  • .shx
  • .dbf
  • .prj

What we are expecting

This tool expects to receive a shapefile with many features, each representing a segment of trail. We expect each of those segments to have attributes about their location, condition, and appropriate use. This tool will keep those fields relevant in OpenTrails, and discard the rest.

Moreover, we expect many segments to only compose a portion of a trail. We will pull out all of the trail names, and create a seperate file called "named_trails.csv", associating each segment with the trail in its entirety.
